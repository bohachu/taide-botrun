# 地雷經驗：修辭手法題目生成經驗

## 問題情境
生成 30 題修辭手法題目（RH-001 到 RH-030），需要涵蓋 10 種必要修辭格，並確保題目難度適中、陷阱設計合理、解析完整。

## 遇到的挑戰

### 1. 修辭格涵蓋的完整性
**挑戰**：需要確保 30 題涵蓋所有必要的修辭格，且分配均衡
**解決方案**：
- 譬喻（明喻、暗喻、借喻）：RH-001, RH-011, RH-015（3題）
- 誇飾：RH-002, RH-012, RH-020（3題）
- 轉化（擬人、擬物）：RH-003, RH-016（2題）
- 排比：RH-004, RH-017, RH-024（3題）
- 映襯（正襯、反襯）：RH-005, RH-018, RH-025（3題）
- 雙關（諧音、語義）：RH-006, RH-013, RH-021, RH-027（4題）
- 借代：RH-007, RH-014, RH-022, RH-028（4題）
- 頂真：RH-008, RH-023（2題）
- 層遞：RH-009, RH-024（2題）
- 設問（提問、激問、懸問）：RH-010, RH-019, RH-026, RH-030（4題）
- 複合修辭：RH-015, RH-020, RH-029（3題）

### 2. 修辭格之間的易混淆點
**挑戰**：許多修辭格容易混淆，需要精確設計陷阱選項
**關鍵區分**：
- **明喻 vs 暗喻 vs 借喻**（RH-001, RH-011）
  - 明喻：有「如、像」等比喻詞
  - 暗喻：用「是」連接本體與喻體
  - 借喻：省略本體，只留喻體

- **排比 vs 對偶 vs 層遞**（RH-004, RH-009, RH-017, RH-024）
  - 排比：≥3個結構相似，並列關係
  - 對偶：2個結構對稱，字數相等
  - 層遞：≥3個有程度遞進或遞降

- **反襯 vs 對比**（RH-005, RH-018）
  - 反襯：有主從關係，一方襯托另一方
  - 對比：兩者並列，地位平等

- **諧音雙關 vs 語義雙關**（RH-006, RH-013, RH-021, RH-027）
  - 諧音雙關：音同字異（如「絲/思」「晴/情」）
  - 語義雙關：一詞多義（如「東風」既是自然風也是助攻風）

- **借代 vs 比喻**（RH-007, RH-014, RH-022, RH-028）
  - 借代：相關性（實際關聯）
  - 比喻：相似性（外表相似）

- **提問 vs 激問 vs 懸問**（RH-010, RH-019, RH-026, RH-030）
  - 提問：自問自答
  - 激問：反問，答案隱含
  - 懸問：有問無答，引發思考

- **擬人 vs 自然描寫**（RH-003, RH-016, RH-029）
  - 擬人：賦予「不該有的」人類特質
  - 自然描寫：事物本來就有的行為

### 3. 陷阱設計的合理性
**挑戰**：陷阱要有迷惑性但不能過於刁鑽
**設計原則**：
- **概念混淆型**：利用相似修辭格的易混點（如明喻誤為暗喻）
- **數量判斷型**：利用數量要求（如排比≥3，對偶=2）
- **不完整分析型**：只看到部分修辭，忽略其他（如「以上皆是」選項）
- **過度解讀型**：把自然現象誤認為修辭（如自然描寫誤為擬人）
- **標誌詞混淆型**：誤判反問標誌詞（如「豈」「安」等）

**每個陷阱都需要**：
- `trap_type`：陷阱類型
- `why_wrong`：為何錯誤
- `how_to_avoid`：如何避免
- `common_mistake_rate`：預估錯誤率（0.10-0.50）

### 4. 例句選擇的經典性
**挑戰**：選擇學生熟悉但不過於簡單的例句
**策略**：
- 優先使用高中課本常見詩詞（唐詩、宋詞、學測常考篇章）
- 避免生僻或爭議性的例句
- 確保例句的修辭判斷沒有爭議性
- 經典例句：
  - 李白：「白髮三千丈」（誇飾）、「人生如逆旅」（明喻+暗喻）
  - 杜甫：「感時花濺淚」（擬人+以樂襯哀）、「國破山河在」（以樂襯哀）
  - 王籍：「蟬噪林逾靜，鳥鳴山更幽」（反襯+對偶）
  - 文天祥：「人生自古誰無死」（激問）
  - 蘇軾：「東風不與周郎便」（語義雙關）

### 5. 複合修辭的處理
**挑戰**：一句話可能同時使用多種修辭，如何完整分析
**處理方式**：
- 設計專門的複合修辭題（RH-015, RH-020, RH-029）
- 要求學生「最完整」的分析，不能只看到一種修辭
- 避免過度解讀：不是所有可能的修辭都要算進去
- 例如 RH-029「蟬噪林逾靜」：
  - ✓ 反襯（以聲襯靜）
  - ✓ 對偶（兩句對稱）
  - ✗ 擬人（蟬鳥本來就會叫，非擬人）
  - ✗ 頂真（「靜」與「鳥」無關聯）

### 6. 難度梯度的控制
**難度分配**：
- Level 3（中等）：15 題（50%）
  - 基本修辭辨識：RH-001, RH-002, RH-004, RH-007, RH-008, RH-016, RH-019, RH-021, RH-026
- Level 4（中等偏難）：13 題（43%）
  - 修辭格區分、類型判斷：RH-005, RH-006, RH-010, RH-011, RH-013, RH-014, RH-017, RH-018, RH-022, RH-023, RH-024, RH-027, RH-028, RH-030
- Level 5（難）：2 題（7%）
  - 複合修辭、精確分析：RH-020, RH-029

**discrimination（鑑別度）**：
- 0.65-0.72：基本題（10 題）
- 0.73-0.78：中等題（15 題）
- 0.79-0.82：難題（5 題）

### 7. 解析品質的要求
**每題必須包含**：
1. **reasoning.steps**（5 步推理）：
   - 理解相關概念
   - 逐一分析各選項
   - 確認正確答案

2. **reasoning.principle**（原理說明）：
   - 該修辭格的定義
   - 判斷標準
   - 與相似修辭格的區別

3. **traps**（陷阱分析）：
   - 每個錯誤選項都要分析
   - 說明錯誤原因
   - 提供避免方法
   - 預估錯誤率

4. **learning**（延伸學習）：
   - 答對：延伸更深入的例子
   - 答錯：複習重點、相關題目推薦

## 成功經驗總結

### 1. 系統性涵蓋
✓ 使用檢查清單確保所有必要修辭格都涵蓋
✓ 每種修辭格至少 2 題，常見易混的多設計幾題
✓ 設計複合修辭題測試綜合能力

### 2. 精確的概念區分
✓ 每種修辭格都明確定義判斷標準
✓ 特別強調易混點的區別（如排比vs對偶vs層遞）
✓ 用經典例句示範正確與錯誤的判斷

### 3. 合理的陷阱設計
✓ 陷阱來自真實的學習困難點
✓ 錯誤選項有迷惑性但不刁鑽
✓ 每個陷阱都有教學價值（幫助學生理解概念）

### 4. 完整的解析內容
✓ 推理步驟清晰完整
✓ 原理說明精確
✓ 陷阱分析有教學價值
✓ 延伸學習有深度

### 5. JSONL 格式的一致性
✓ 所有題目都符合 schema 要求
✓ 每題一個 JSONL 檔案
✓ 檔名格式：{id}.jsonl
✓ 內容為單行 JSON

## 建議的題目生成流程

1. **規劃階段**（10%時間）：
   - 列出所有必要涵蓋的修辭格
   - 規劃每種修辭格的題數
   - 確定難度分配

2. **設計階段**（40%時間）：
   - 選擇經典例句
   - 設計題幹與選項
   - 設計合理的陷阱
   - 確定正確答案

3. **解析階段**（40%時間）：
   - 撰寫推理步驟
   - 說明原理與判斷標準
   - 分析每個陷阱
   - 設計延伸學習

4. **檢查階段**（10%時間）：
   - 確認涵蓋完整性
   - 檢查 JSONL 格式
   - 驗證答案正確性
   - 確認解析品質
